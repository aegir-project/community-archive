How to welcome someone to the core team

Updated by anarcat on 04/11/2013 - 21:36
AddedChangedDeleted

Once someone is deemed worthy of joining the core team, you (a core member) should make a proposal on the core mailing list to welcome the new developer. Subject should clearly state the proposal and give ample time for other developers to chime in. Usually all core team members should give their explicit approval as we try to reach consensus (and usually do) over proposals.

Once the member is approved on the mailing list, you should give the following access:

  • membership on the mailing list
  • commit access to the core repositories (Hostmaster, Hosting, Provision and Eldir)
  • jenkins access (simply create a user then go to the configuration page to give him the same access as others - consider also Vagrant scripts used in Jenkins and maybe the Aegir Jenkins scripts
  • operator status on the IRC channel

Then send a nice welcome email to the new member and CC the mailing list. Optionnally announce the great news in the next release.
