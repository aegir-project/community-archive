strange issue with first time login for new sites

Tagged:
  • first time login
  • goto_site

Hi - I'm having a problem (just recently) where the first time logins for new sites are not redirecting to the first time login url.

When they click on their 'go to site' link, they are being redirected to the main page of the aegir install.

If the user then goes back to that link and click it again they ARE sent to the site -- but the first time login is gone.

The only resolution is to do a reset password immediately after a new site is created. If that happens the user is successfully redirected to their site and can create their credentials.

The admin user does not have this problem. They get a correctly formed first time login on a new site creation.

Any random thoughts on why this might be happening?

Thanks, KC
